## Теоретическая часть

### 1. Алгоритмы сортировки

**Пузырьковая сортировка (Bubble Sort)**  
- Принцип работы: сравнивает соседние элементы и перемещает больший элемент вправо.  
- Время выполнения: O(n²).  
- Подходит для небольших массивов или частично отсортированных данных.  

**Сортировка выбором (Selection Sort)**  
- Принцип работы: находит минимальный элемент и перемещает его в начало массива.  
- Время выполнения: O(n²).  
- Меньше перемещений по сравнению с пузырьком, но также менее эффективна на больших массивах.  

**Сортировка вставками (Insertion Sort)**  
- Принцип работы: элементы последовательно перемещаются в отсортированную часть массива.  
- Время выполнения: O(n²), но эффективна на небольших или частично отсортированных массивах.  

### 2. Параллельное программирование с OpenMP

- OpenMP — API для параллельного программирования на CPU.  
- Директивы, используемые в работе:  
  - `#pragma omp parallel for` — распараллеливание циклов.  
  - `reduction` — суммирование значений из разных потоков.  
  - `#pragma omp critical` — защита критических участков.  
- Некоторые алгоритмы, например сортировка вставками, трудно распараллеливать из-за последовательных зависимостей.

### 3. Особенности параллельной реализации сортировок

- **Пузырьковая сортировка**: внешние циклы можно распараллелить, важно избегать конфликтов при обмене соседних элементов.  
- **Сортировка выбором**: каждый поток может искать минимальный элемент в своей части массива.  
- **Сортировка вставками**: менее подходит для параллельной реализации; применяется частичный параллелизм или Shell-сортировка.
